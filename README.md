# Blogicum

Бэкенд-часть веб-приложения блога на Django. Платформа для публикации постов с поддержкой категорий, комментариев, профилей пользователей и управления контентом.

## Описание

Backend-приложение предоставляет RESTful API и серверную логику для блог-платформы. Реализована система публикаций с отложенной публикацией, категоризацией контента, комментариями, управлением пользовательскими профилями и загрузкой медиа-файлов.

## Основной функционал

- **Управление публикациями**: создание, редактирование, удаление постов с поддержкой отложенной публикации
- **Система комментариев**: добавление, редактирование и удаление комментариев к публикациям
- **Категории и местоположения**: организация контента по тематическим категориям и географическим меткам
- **Профили пользователей**: управление профилями с возможностью редактирования данных
- **Аутентификация и авторизация**: регистрация, вход, проверка прав доступа к контенту
- **Загрузка изображений**: поддержка медиа-файлов для публикаций
- **Пагинация**: эффективная навигация по спискам постов и комментариев
- **Административная панель**: управление контентом через Django Admin

## Технологический стек

### Backend
- **Django 5.1.1** — основной фреймворк
- **SQLite** — база данных (для разработки)
- **Pillow 11.0.0** — обработка изображений
- **django-bootstrap5 24.3** — интеграция Bootstrap для форм
- **django-debug-toolbar 5.1.0** — инструменты отладки

### Инструменты разработки
- **pytest 8.3.3** + **pytest-django 4.9.0** — тестирование
- **flake8 7.1.1** — линтинг кода
- **yapf 0.43.0** — форматирование кода
- **Faker 12.0.1** — генерация тестовых данных
- **mixer 7.2.2** — создание тестовых фикстур

### Frontend
- Предоставлен отдельно (Bootstrap, HTML/CSS)

## Архитектура

Проект организован по принципу Django apps:
- `blog` — основное приложение с моделями постов, комментариев, категорий и представлениями
- `pages` — статические страницы и обработчики ошибок

### Ключевые компоненты

- **Модели**: `Post`, `Comment`, `Category`, `Location` с кастомными QuerySet и менеджерами
- **Представления**: класс-базированные (CBV) и функциональные представления
- **Формы**: валидация данных для постов, комментариев и профилей
- **Mixins**: переиспользуемая логика проверки прав доступа (`AuthorCheckMixin`)
- **Сервисы**: утилиты для пагинации и обработки текста
